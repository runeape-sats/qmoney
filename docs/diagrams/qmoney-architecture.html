QMoney Architecture

Current repo split: pkey_quorum baseline, pubkey_hidden_subspace prototype, and formal/spec verification layers.

Baseline logic

  • • `pkey_quorum/demo.py` is the canonical private-key cash baseline.
  • • Verification is destructive and remints a fresh note for the receiver.
  • • The main fixed safety property is: no spend if quorum cannot assemble.

Research track

  • • `pubkey_hidden_subspace/note_family.py` is intentionally separate from BB84/quorum logic.
  • • It mints a uniform superposition over A and checks the dual-space condition in A⊥.
  • • Current scope is tiny-n toy simulation, not cryptographic obfuscation.

Verification layers

  • • TLA+ captures the abstract issuance / oracle / accept-reject lifecycle.
  • • TLC already validated the current oracle invariants with bounded queries.
  • • Z3 is the current layer for inter-state inductive stability checks.
